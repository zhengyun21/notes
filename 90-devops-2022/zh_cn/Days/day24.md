## 网络自动化

### 网络自动化基础

网络自动化的主要驱动力：
- 提升敏捷性
- 降低成本
- 消除错误
- 确保合规
- 集中管理

自动化的落地过程因企业而异。谈到部署自动化，并不存在放之四海而皆准的方案。能够识别并采用最适合你组织的方法，对于推进、维持或创建更敏捷的环境至关重要，关注点始终应该放在业务价值和最终用户体验上。（我们在一开始谈 DevOps、文化变革以及它带来的自动化流程时，也提到过类似的观点。）

要拆解这个问题，你需要识别你想自动化的任务或流程将如何在遵循逐步、系统化方法的同时，提升最终用户体验或业务价值。

"如果你不知道自己要去哪里，那么走哪条路都一样。"

先明确你想达成的框架或设计结构，知道最终目标是什么，再一步一步朝着目标前进，并根据业务结果在不同阶段衡量自动化是否成功。

围绕现有应用来构建概念模型。没有必要在真空中设计自动化概念，因为它们最终需要应用到你的应用、服务和基础设施上，所以应当从你现有的基础设施和现有应用出发来构建这些概念。

### 网络自动化的方法

我们应该识别这些任务，并对网络变更请求进行调研，这样就能找出最常见的问题，然后把解决方案自动化。

- 列出所有目前仍在手动处理的变更请求和工作流。
- 确定最常见、最耗时、最容易出错的活动。
- 采用以业务为驱动的方法对这些请求进行优先级排序。
- 这就是构建自动化流程的框架，也就是哪些必须自动化，哪些不必自动化。

接下来，我们应该对任务进行拆分，并分析不同网络功能如何协同工作、彼此交互。

- 基础设施/网络团队会在多个层面接收变更工单，以部署应用。
- 根据网络服务，将其划分为不同领域，并理解它们如何相互作用。
    - 应用优化
    - ADC（应用交付控制器）
    - 防火墙
    - DDI（DNS、DHCP、IPAM 等）
    - 路由
    - 其他
- 识别各种依赖关系，以处理业务和文化上的差异，并引入跨团队协作。

可复用的策略，定义并简化可复用的服务任务、流程以及输入/输出。

- 为各类服务、流程和输入/输出定义标准化能力。
- 简化部署流程将缩短新旧工作负载的上市时间。
- 一旦你有了标准流程，就可以对其进行排序，并将其与单个请求对齐，从而实现多线程式的处理和交付。

将这些策略与具体业务活动结合起来。实施这项策略如何帮助业务？节省时间？节省成本？带来更好的业务结果？

- 确保服务任务之间具备互操作性。
- 将增量式服务任务关联起来，使其对齐并组合成业务服务。
- 允许按需灵活地关联和重新关联服务任务。
- 部署自助服务能力，为提升运营效率铺平道路。
- 让多种技术技能继续在监督和合规的前提下发挥作用。

对策略和流程进行**迭代**，在保持可用性和服务质量的同时不断增加和改进。

- 从自动化现有任务开始，先从小处着手。
- 熟悉自动化流程，这样你就能识别出哪些其他领域也能从自动化中受益。
- 逐步迭代你的自动化举措，在保持所需可用性的同时渐进式地增强敏捷性。
- 采取渐进式方法会为成功铺平道路！

编排网络服务！

- 要快速交付应用，就需要自动化部署流程。
- 创建敏捷的服务环境，需要在不同技术技能之间管理各种要素。
- 准备好端到端的编排，以便对自动化和部署顺序进行控制。

## 网络自动化工具

好消息是，在大多数情况下，我们用于网络自动化的工具，通常也与其他自动化领域所使用的工具相同，或者至少与我们到目前为止已经讲过、以及未来还会讲到的工具相同。

操作系统 - 正如我在整个挑战中一直做的那样，我主要使用 Linux 操作系统来学习。原因我在 Linux 部分已经说明过了。我们接下来会接触到的几乎所有工具，尽管现在大多支持跨平台，但最初都起源于 Linux 应用或工具。

集成开发环境（IDE） - 这里也没太多可说的，我会一如既往地推荐 Visual Studio Code 作为你的 IDE，因为它支持非常多语言，并且有大量可用插件。

配置管理 - 我们还没有讲到配置管理部分，但很明显，Ansible 是这一领域中用于管理和自动化配置的热门工具。Ansible 是用 Python 编写的，但你不需要先懂 Python。
    
- 无需代理
- 只需要 SSH
- 社区支持广泛
- 拥有大量网络模块
- 仅推送模式
- 使用 YAML 配置
- 开源！

[Ansible 网络模块链接](https://docs.ansible.com/ansible/2.9/modules/list_of_network_modules.html)

我们也会在配置管理部分提到 **Ansible Tower**，可以把它看作 Ansible 的图形界面前端。

CI/CD - 关于这部分的概念和工具，我们后面还会更深入地讲，但这里至少要先提一下，因为它不仅涉及网络，也涉及所有服务和平台的资源交付。

尤其是 Jenkins，在网络自动化领域似乎是一个很受欢迎的工具。

- 监控 Git 仓库中的变更，然后触发相应流程。

版本控制 - 这也是我们后面会更深入讨论的内容。

- Git 为你本地设备上的代码提供版本控制，且支持跨平台。
- GitHub、GitLab、Bitbucket 等在线网站则用于定义仓库并上传代码。

语言 | 脚本 - 我们这里还没有讲到 Python 这门语言。我根据自己的情况选择了 Go 作为编程语言来深入学习。我会说，Golang 和 Python 的竞争非常接近，但在网络自动化领域，Python 看起来是赢家。

- 这里还要提一下 Nornir，这是一个用 Python 编写的自动化框架。它似乎承担了类似 Ansible 的角色，但更专注于网络自动化。[Nornir 文档](https://nornir.readthedocs.io/en/latest/)

API 分析 - Postman 是分析 RESTful API 的绝佳工具。它有助于构建、测试和修改 API。

- POST >>> 用于创建资源对象。
- GET  >>> 用于获取资源。
- PUT  >>> 用于创建或替换资源。
- PATCH >>> 用于创建或更新资源对象。
- DELETE >>> 用于删除资源。

[Postman 工具下载](https://www.postman.com/downloads/)
