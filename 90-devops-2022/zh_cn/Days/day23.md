## 网络协议

一组构成标准的规则和消息，也就是一个互联网标准。

- ARP - 地址解析协议

如果你想深入了解 ARP，可以在这里阅读互联网标准。[RFC 826](https://datatracker.ietf.org/doc/html/rfc826)

它把 IP 地址和固定的物理机器地址，也就是 MAC 地址，连接起来，作用于第 2 层网络中。

![](Images/Day23_Networking1.png)

- FTP - 文件传输协议

用于将文件从源端传输到目标端。通常这个过程需要身份验证，但如果配置允许，也可以使用匿名访问。你现在更常见到的是 FTPS，它为客户端到 FTP 服务器的连接提供 SSL/TLS，从而提升安全性。这个协议会出现在 OSI 模型的应用层。

![](Images/Day23_Networking2.png)

- SMTP - 简单邮件传输协议

用于电子邮件传输，邮件服务器使用 SMTP 来发送和接收邮件消息。即使在 Microsoft 365 中，你仍然会发现 SMTP 协议被用于同样的目的。

![](Images/Day23_Networking3.png)

- HTTP - 超文本传输协议

HTTP 是互联网和网页内容浏览的基础。它让我们能够轻松访问自己喜欢的网站。HTTP 仍然被广泛使用，但在大多数你喜欢的网站上，HTTPS 使用得更多，或者说应该更多地使用 HTTPS。

![](Images/Day23_Networking4.png)

- SSL - 安全套接字层 | TLS - 传输层安全

TLS 已经取代了 SSL。TLS 是一种加密协议，用于在网络上提供安全通信。它可以出现在邮件、即时通讯和其他应用中，但最常见的用途还是保护 HTTPS。

![](Images/Day23_Networking5.png)

- HTTPS - 使用 SSL/TLS 保护的 HTTP

它是 HTTP 的扩展，用于在网络上进行安全通信。正如前面提到的，HTTPS 通过 TLS 进行加密。这里的重点是，在主机之间交换数据时，提供认证、隐私和完整性。

![](Images/Day23_Networking6.png)

- DNS - 域名系统

DNS 用于把人类更容易记住的域名映射到 IP 地址。比如我们都知道 [google.com](https://google.com)，但如果你在浏览器里输入 [8.8.8.8](https://8.8.8.8)，你看到的基本上还是 Google。不过，要记住你所有网站的 IP 地址可就没那么轻松了，而其中一些信息我们甚至还是通过 Google 去查找的。

这就是 DNS 发挥作用的地方。它确保主机、服务和其他资源可以被访问。

对于所有需要接入互联网的主机来说，它们必须具备 DNS，才能解析这些域名。DNS 是一个你完全可以花上数天甚至数年来学习的领域。根据我的经验，我也会说 DNS 往往是网络问题中最常见的错误来源。不知道网络工程师是否也会这么认同。

![](Images/Day23_Networking7.png)

- DHCP - 动态主机配置协议

前面我们已经讨论了很多让主机正常工作的协议，不管是访问互联网，还是在彼此之间传输文件。

每一台主机都需要 4 个东西，才能完成这两类任务。

- IP 地址
- 子网掩码
- 默认网关
- DNS

我们已经讲过 IP 地址，它是主机在所处网络中的唯一地址。可以把它想成门牌号。

子网掩码我们很快会讲到，不过你可以把它想成邮编或 ZIP 码。

默认网关是我们网络中路由器的 IP，通常它为我们提供第 3 层连通性。你可以把它看作那条能让我们离开街区的唯一道路。

然后是 DNS，正如刚才讲过的，它帮助我们把复杂的公网 IP 地址转换成更合适、更容易记住的域名。也许我们可以把它想成一个巨大的分拣中心，确保邮件送到正确的地方。

正如我所说，每台主机都需要这 4 项配置。如果你有 1000 台甚至 10000 台主机，那么逐个手动配置会花很长时间。这就是 DHCP 的作用，它允许你为网络定义一个地址池，然后由这个协议把地址分发给网络中的所有可用主机。

再举个例子，你走进一家咖啡店，买了杯咖啡，然后拿出笔记本电脑或手机，我们把它叫作你的主机。你把主机连上咖啡店的 Wi-Fi，就可以上网，消息和邮件开始不断到来，你也能浏览网页和社交媒体。你连接咖啡店 Wi-Fi 时，设备会从专用 DHCP 服务器获取一个地址，或者更常见地，从同时负责 DHCP 的路由器那里获取地址。

### 子网划分

子网是 IP 网络的一个逻辑子划分。

子网把大型网络拆分成更小、更易管理的网络，从而让它们运行得更高效。

每个子网都是更大网络中的一个逻辑子集。连接在一起、并且拥有足够子网条件的设备共享相同的 IP 地址标识符，从而能够彼此通信。

路由器负责管理子网之间的通信。

子网的大小取决于连接需求和所使用的网络技术。

组织需要在可用地址空间的限制内，决定子网的数量和大小，而这些细节都由该组织自行本地管理。子网也可以进一步细分为更小的子网，比如点对点链路，或者支持少量设备的子网络。

除了其他优势之外，把大型网络划分为子网还能实现 IP 地址的重新分配，并缓解网络拥塞，让网络通信更加顺畅、效率更高。

子网也能提升网络安全性。如果网络中的某个区域遭到入侵，就可以将其隔离，从而让攻击者更难在更大的网络中横向移动。
