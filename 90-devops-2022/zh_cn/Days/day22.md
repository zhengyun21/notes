## OSI 模型 - 7 层

网络这门行业的总体目标，是让两台主机能够共享数据。若是在有网络之前，我想把数据从这台主机传到那台主机，就得先把某种东西插到这台主机上，再带到另一台主机上，然后再插进去。

网络让这一切自动化了，让主机能够通过线路自动共享数据。要做到这一点，这些主机必须遵循一套规则。

这和任何语言都没什么不同。英语有一套规则，两个人要想用英语交流就必须遵守；西班牙语也有自己的规则；法语也有自己的规则。网络同样也有自己的规则。

网络规则被划分为七个不同的层，这七层就是 OSI 模型。

### OSI 模型简介

OSI 模型（Open Systems Interconnection Model，开放系统互连模型）是一个用于描述网络系统功能的框架。OSI 模型将计算功能划分为一套通用的规则和要求，以支持不同产品和软件之间的互操作性。在 OSI 参考模型中，计算系统之间的通信被拆分为七个不同的抽象层：**物理层、数据链路层、网络层、传输层、会话层、表示层和应用层**。

![](Images/Day22_Networking1.png)

### 物理层

OSI 模型的第 1 层，也就是物理层。它的前提是能够通过某种方式把数据从一台主机传到另一台主机，这种方式可以是物理网线，我们也可以把 Wi-Fi 归到这一层。在这里我们还可能看到一些较为传统的硬件，比如集线器和中继器，用来把数据从一台主机传到另一台主机。

![](Images/Day22_Networking2.png)

### 数据链路层

第 2 层，数据链路层，支持节点到节点的数据传输，数据会被封装成帧。这里也会有一层错误修正，可能用于处理物理层中发生的错误。这里也是我们第一次引入或看到 MAC 地址的地方。

这也是我们第一次提到交换机，它们在我们的第 21 天网络基础中已经讲过，见 [Day 21](day21.md)

![](Images/Day22_Networking3.png)

### 网络层

你可能听过三层交换机或二层交换机这样的说法。在 OSI 模型中，第 3 层是网络层，其目标是端到端交付，这里我们也会再次看到之前概览里提到的 IP 地址。

路由器和主机都存在于第 3 层。记住，路由器的作用是能够在多个网络之间进行路由。任何带有 IP 的设备都可以被认为是第 3 层设备。

![](Images/Day22_Networking4.png)

那么为什么我们同时需要第 2 层和第 3 层的寻址方式呢？（MAC 地址 vs IP 地址）

如果我们考虑把数据从一台主机传到另一台主机，每台主机都有一个 IP 地址，但中间还会经过多个交换机和路由器。每个设备都有自己的第 2 层 MAC 地址。

第 2 层的 MAC 地址只会在主机到交换机/路由器这一跳中使用，它关注的是每一跳；而第 3 层的 IP 地址会一直跟随这份数据包，直到它到达最终主机。（端到端）

IP 地址 - 第 3 层 = 端到端交付

MAC 地址 - 第 2 层 = 一跳一跳交付

这里还有一个网络协议我们之后会讲到，但今天不展开，它叫 ARP（Address Resolution Protocol，地址解析协议），它把第 3 层和第 2 层的地址关联起来。

### 传输层

服务到服务的交付。第 4 层用来区分数据流。和第 3 层与第 2 层都有各自的寻址方式一样，第 4 层使用的是端口。

![](Images/Day22_Networking5.png)

### 会话层、表示层、应用层

第 5、6、7 层之间的区分已经变得有些模糊。

你也可以看看 [TCP IP 模型](https://www.geeksforgeeks.org/tcp-ip-model/)，以便获得更贴近现代的理解。

现在让我们试着解释一下，当主机通过这套网络栈彼此通信时，实际发生了什么。这个主机上有一个应用，它会生成要发送到另一台主机的数据。

源主机会经历所谓的封装过程。数据首先会被发送到第 4 层。

第 4 层会给这些数据加上一个头部，以实现第 4 层的目标，也就是服务到服务交付。这里会使用 TCP 或 UDP 的端口，同时还会包含源端口和目的端口。

这也可能被称为一个段（数据 + 端口）。

这个段会继续沿着 OSI 栈向下传到第 3 层，也就是网络层。网络层会再次给这份数据添加一个头部。
这个头部会实现第 3 层的目标，也就是端到端交付，这意味着在这个头部里会包含源 IP 地址和目的 IP 地址；头部加上数据也可能被称为一个包。

随后第 3 层会把这个包交给第 2 层，第 2 层会再次给这份数据添加一个头部，以实现第 2 层一跳一跳交付的目标，这意味着这个头部会包含源 MAC 地址和目的 MAC 地址。
当你有第 2 层头部和数据时，这就叫做帧。

然后这个帧会被转换成 0 和 1，并通过第 1 层的物理网线或 Wi-Fi 发送出去。

![](Images/Day22_Networking6.png)
