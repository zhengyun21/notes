# 16 - 单元测试

## 1. Go 的测试工具

Go 自带测试框架 `testing`，使用 `go test` 运行测试。

测试文件规则：

- 文件名以 **`_test.go`** 结尾。
- 测试函数以 **`Test`** 开头。
- 测试函数参数是 `t *testing.T`。

## 2. 被测试代码

示例文件：`calc.go`

```go
package main

func Add(a, b int) int {
	return a + b
}
```

## 3. 基础测试

示例文件：`calc_test.go`

```go
package main

import "testing"

func TestAdd(t *testing.T) {
	got := Add(1, 2)
	want := 3

	if got != want {
		t.Fatalf("Add(1, 2) = %d, want %d", got, want)
	}
}
```

运行：

```powershell
go test
```

**重点：**测试失败时用 `t.Fatal` 或 `t.Fatalf` 终止当前测试。

## 4. 表格驱动测试

多个输入输出场景，推荐用表格驱动测试。

```go
package main

import "testing"

func TestAdd(t *testing.T) {
	tests := []struct {
		name string
		a    int
		b    int
		want int
	}{
		{name: "positive", a: 1, b: 2, want: 3},
		{name: "zero", a: 0, b: 0, want: 0},
		{name: "negative", a: -1, b: -2, want: -3},
	}

	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			got := Add(tt.a, tt.b)
			if got != tt.want {
				t.Fatalf("got %d, want %d", got, tt.want)
			}
		})
	}
}
```

**重点：**`t.Run` 可以给每组测试起名字，失败时更容易定位。

## 5. 运行指定测试

只运行某个测试：

```powershell
go test -run TestAdd
```

显示详细过程：

```powershell
go test -v
```

运行当前目录和子目录：

```powershell
go test ./...
```

## 6. 性能测试

性能测试函数以 `Benchmark` 开头，参数是 `b *testing.B`。

```go
package main

import "testing"

func BenchmarkAdd(b *testing.B) {
	for i := 0; i < b.N; i++ {
		Add(1, 2)
	}
}
```

运行：

```powershell
go test -bench=.
```

**重点：**`b.N` 由 Go 自动调整，不需要自己指定。

## 7. 易错点

- **测试文件必须以 `_test.go` 结尾**。
- **测试函数必须以 `Test` 开头**。
- **测试函数签名必须是 `func TestXxx(t *testing.T)`**。
- **性能测试函数必须以 `Benchmark` 开头**。
- **多个用例优先写表格驱动测试**。
- **提交前优先运行 `go test ./...`**。
