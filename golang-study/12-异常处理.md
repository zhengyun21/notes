# 12 - 异常处理

## 1. Go 怎么处理错误

Go 没有传统的 `try catch`。常见做法是：函数返回 `error`，调用方自己判断。

```go
if err != nil {
	return err
}
```

**重点：**Go 里错误通常是普通返回值，不是异常。

## 2. 返回 error

函数执行失败时，可以返回 `error`。

```go
package main

import (
	"errors"
	"fmt"
)

func checkAge(age int) error {
	if age < 0 {
		return errors.New("年龄不能小于 0")
	}

	return nil
}

func main() {
	err := checkAge(-1)
	fmt.Println(err)
}
```

**重点：**没有错误时返回 `nil`。

## 3. 向上返回错误

当前函数不知道怎么处理错误时，可以把错误继续返回给上层。

```go
package main

import (
	"errors"
	"fmt"
)

func readConfig() error {
	return errors.New("配置读取失败")
}

func startApp() error {
	err := readConfig()
	if err != nil {
		return err
	}

	return nil
}

func main() {
	err := startApp()
	if err != nil {
		fmt.Println("启动失败：", err)
	}
}
```

**注意：**不要在底层函数里随便决定程序要不要退出。

## 4. 包装错误

返回错误时，可以补充上下文。

```go
package main

import (
	"errors"
	"fmt"
)

func readConfig() error {
	return errors.New("文件不存在")
}

func startApp() error {
	if err := readConfig(); err != nil {
		return fmt.Errorf("读取配置失败：%w", err)
	}

	return nil
}

func main() {
	fmt.Println(startApp())
}
```

**重点：**`%w` 会保留原始错误，方便后续判断。

## 5. panic

`panic` 会中断当前正常流程，通常用于**程序无法继续运行**的场景。

```go
package main

import "fmt"

func main() {
	panic("程序出错")

	fmt.Println("不会执行")
}
```

常见场景：

- 初始化失败。
- 必需配置不存在。
- 程序进入不该出现的状态。

**注意：**普通业务错误不要动不动就 `panic`。

## 6. 初始化失败时中断程序

初始化阶段失败，继续运行可能没有意义，可以直接停止。

```go
package main

import (
	"fmt"
	"os"
)

func init() {
	_, err := os.ReadFile("config.json")
	if err != nil {
		panic(err)
	}
}

func main() {
	fmt.Println("程序启动")
}
```

**重点：**`panic` 更适合处理不可恢复错误。

## 7. recover

`recover` 可以在 `defer` 中捕获 `panic`，避免程序直接崩溃。

```go
package main

import "fmt"

func safeRun() {
	defer func() {
		if err := recover(); err != nil {
			fmt.Println("捕获 panic：", err)
		}
	}()

	panic("出错了")
}

func main() {
	safeRun()
	fmt.Println("程序继续执行")
}
```

**重点：**`recover` 必须在 `defer` 调用的函数中使用。

## 8. 打印堆栈信息

调试时可以用 `debug.Stack()` 打印调用堆栈。

```go
package main

import (
	"fmt"
	"runtime/debug"
)

func safeRun() {
	defer func() {
		if err := recover(); err != nil {
			fmt.Println(err)
			fmt.Println(string(debug.Stack()))
		}
	}()

	list := []int{1, 2}
	fmt.Println(list[2])
}

func main() {
	safeRun()
}
```

**注意：**堆栈适合排查问题，正式输出里不要随便暴露。

## 9. error、panic、recover 怎么选

简单记：

- **可预期错误**：返回 `error`。
- **无法继续运行**：使用 `panic`。
- **框架或最外层兜底**：使用 `recover`。

## 10. 易错点

- **Go 没有 `try catch`**。
- **普通错误优先返回 `error`**。
- **没有错误时返回 `nil`**。
- **底层函数不要轻易退出程序**。
- **`panic` 适合不可恢复错误**。
- **`recover` 必须配合 `defer` 使用**。
- **`%w` 可以包装并保留原始错误**。
