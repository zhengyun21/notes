1. 发布一个版本时，我们通常先在版本库中打一个标签（tag），这样，就唯一确定了打标签时刻的版本。将来无论什么时候，取某个标签的版本，就是把那个打标签的时刻的历史版本取出来。所以，标签也是版本库的一个快照。
2. Git的标签虽然是版本库的快照，但其实它就是指向某个commit的指针（跟分支很像对不对？但是分支可以移动，标签不能移动），所以，创建和删除标签都是瞬间完成的。
3. 创建标签
	1. 切换到要打标签的分支
	2. git tag < name >
	3. 如果前面忘记打标签，就找到对应的 commit id 进行打标签，比方说要对`add merge`这次提交打标签，它对应的commit id是`f52c633`，敲入命令：```plain$ git tag v0.9 f52c633```
	4. git tag 查看所有标签
4. 操作标签
	1. 删除标签 ```git tag -d v0.1```
	2. 推送某个标签到远程，使用命令`git push origin <tagname>`
