# 运维练手项目指南

> 本文档整理了 GitHub 上适合练手的运维/DevOps 项目，以及做项目前需要掌握的前置技能，方便按图索骥、循序渐进。

---

## 一、GitHub 现成运维/DevOps 练手项目

按 **新手入门 → 进阶容器 CI/CD → K8s 微服务 → 完整运维平台** 分级，全部带完整部署文档、Docker/Compose/GitHub Actions，拿来就能练从 0 到 1 上线全流程。

### 1. 新手入门（零基础，单服务器 Docker + GitHub Actions）

#### 1.1 flask-devops-demo（极简全链路，推荐第一个练）

- **仓库**：https://github.com/ruppysuppy/flask-devops-demo
- **技术栈**：Python Flask + Docker + GitHub Actions + DockerHub + SSH 自动部署 ECS
- **练手点**：
  1. 本地 Dockerfile 打包、本地 compose 调试
  2. Actions 自动单元测试 → 构建镜像 → 推送镜像仓库
  3. SSH 远程连接服务器自动拉取镜像重启服务
  4. 环境变量分离、日志挂载、容器自重启
- **优势**：代码极少、注释完整、中文教程多，一台轻量云服务器就能跑通整套 CI/CD。

#### 1.2 azure-voting-app-redis（微软官方标准演示）

- **仓库**：https://github.com/Azure-Samples/azure-voting-app-redis
- **架构**：前端 Web + Redis 缓存，多容器依赖
- **练手**：Docker Compose 多服务编排、网络互通、数据持久卷、CI 流水线、可扩展到 K8s
- **适合**：练习多组件联动、容器依赖顺序、数据持久化。

#### 1.3 nodejs-docker-cicd-demo（前端/Node 后端运维）

- **仓库**：https://github.com/stevensona/node-docker-github-actions
- **内容**：Node API + Nginx 反向代理、多环境 env、一键 deploy 脚本、Actions 自动发布到阿里云 ECS
- **适合**想练 **前端静态资源部署、Nginx 反向代理** 的运维。

### 2. 进阶：前后端分离完整 DevOps 项目

#### 2.1 vue-flask-devops（前后端分离全套运维模板）

- **仓库**：https://github.com/chengyangfu/vue-flask-devops
- **目录自带**：
  - frontend/vue3、backend/flask
  - docker-compose.dev / docker-compose.prod 两套环境
  - .github/workflows CI/CD：构建前端静态镜像、后端镜像、SSH 远程部署脚本
  - deploy/ 目录：服务器初始化脚本、备份脚本、回滚脚本
- **练手内容**：多环境隔离、前端打包优化、Nginx 动静分离、生产资源限制、自动清理镜像。

#### 2.2 devops-project1（Django 全栈 + Jenkins/GitHub Actions 双流水线）

- **仓库**：https://github.com/T-EJ/Devops-project1
- **两种自动化方案任选**：GitHub Actions 无服务、Jenkins 自建 CI 服务器
- **配套**：MySQL 容器持久化、定时数据库备份脚本、服务器监控 shell
- **适合**同时练 **GitHub 原生 CI** 和 **传统 Jenkins** 两套运维方案。

## 二、做项目前需要掌握的前置技能

### 1. 基础 Linux（所有运维项目根基，必须先吃透）

#### 必掌握知识点

**系统基础**
- 用户/权限：`useradd`、`chmod`、`chown`、`sudo`、免密 ssh
- 文件操作：`cd` `ls` `cp` `mv` `rm` `tar`、软硬链接、`find` 检索日志
- 进程：`ps` `top` `kill` `netstat` `ss` `lsof`，端口占用排查

**服务与网络**
- systemd：`systemctl start/enable/status` 自定义服务
- 防火墙：firewalld/iptables 放行端口
- SSH：密钥登录、`ssh-copy-id`、远程执行命令

**日常运维脚本基础**
- Shell 基础：变量、if 判断、for 循环、函数、读取文件
- 定时任务 crontab、日志切割、备份脚本

**配套工具**
- vim/nano 编辑文件、yum/apt 安装软件、磁盘挂载 `df` `du`

> 💡 达不到的痛点：连服务器都登不稳、端口不通不会排查、不会写简单备份脚本，项目部署直接卡壳。

---

### 2. Git & GitHub 基础（CI/CD 前提）

- **本地 Git 操作**：`git clone add commit push pull branch` 分支切换、简单合并
- **GitHub 核心操作**：Fork 仓库、仓库 Secrets 配置（存放服务器密钥、镜像账号）
- **看懂 .github/workflows**：目录结构，理解触发条件（push/pull_request）
- **基础概念**：仓库、远程仓库、密钥认证、私有仓库权限

> 💡 达不到的痛点：没法拉代码、不会配置流水线秘钥，CI/CD 流水线完全跑不起来。

---

### 3. 容器基础 Docker（90% 练手项目核心）

**必学内容**
- 核心命令：`pull` `run` `ps` `exec` `logs` `rm` `rmi` `volume`
- Dockerfile 编写：`FROM` `COPY` `RUN` `CMD` `ENV` `WORKDIR` `EXPOSE`
- 数据持久：volume 挂载、绑定宿主机目录
- Docker Compose：yaml 语法，多容器编排、依赖、环境变量、网络
- 镜像仓库：DockerHub 登录、构建推送镜像

> 💡 达不到的痛点：不会打包程序、不会多容器联动（web+数据库），项目无法容器化部署。

---

### 4. 网络与 Nginx（线上项目必备反向代理）

- **Nginx 基础配置**：静态资源托管、反向代理、端口转发、负载均衡简单配置
- **基础网络概念**：IP、端口、域名、本地/容器网络、跨机器访问
- **日志查看**：nginx access.log error.log 排错

---

### 5. CI/CD 基础（GitHub Actions 流水线必备）

- 看懂 workflow yaml：`jobs`、`steps`、`runs-on`、`uses`
- 基础 action 使用：检出代码、登录镜像仓库、ssh 远程服务器
- 环境变量、秘钥引用、流水线失败排查思路

---

### 6. 进阶方向（K8s 微服务项目额外需要）

如果后面要玩 K8s 相关练手项目，在上面基础上再加：

1. K8s 核心资源：Pod、Deployment、Service、Ingress、ConfigMap、Secret
2. kubectl 基础命令、yaml 编写
3. 集群基础概念：节点、命名空间、滚动更新、扩容缩容
4. Helm 简单使用

## 四、最低门槛

能跑通最简单运维 demo 的最低要求：
- 会 ssh 登录服务器
- 写简单 Dockerfile
- 会 git 推送代码
- 看得懂基础 Actions 流水线
- 会用 Nginx 反向代理

---

## 五、新手练手项目顺序建议

1. 先 Fork **flask-devops-demo**：吃透 Dockerfile、GitHub Actions、SSH 自动部署
2. 切换 **vue-flask-devops**：练习前后端分离、多环境 compose、Nginx 反向代理
3. 可选升级 **azure-voting-app**：多容器依赖、持久卷
4. 有 K8s 环境再上 k8s 流水线项目

---

## 六、通用实操步骤（所有项目通用）

1. Fork 仓库到自己 GitHub 账号
2. 本地克隆修改代码
3. 云服务器放行 80/22 端口，免密 SSH
4. GitHub 仓库添加 Secrets：Docker 账号、服务器 IP、SSH 私钥
5. 执行服务器一键初始化脚本（install_env.sh）
6. push 代码触发 Actions 自动构建 + 远程部署
7. 练习手动回滚、扩容、日志排查、备份恢复
