# 15 - 自定义数据类型与类型别名

## 1. 自定义类型是什么

使用 `type` 可以定义一个新类型。

```go
type Code int
```

`Code` 的底层类型是 `int`，但它已经是一个**新类型**。

**重点：**自定义类型可以让代码含义更清楚。

## 2. 自定义类型示例

用自定义类型表示响应码。

```go
package main

import "fmt"

type Code int

const (
	SuccessCode Code = 0
	ValidCode   Code = 7
	ServerCode  Code = 8
)

func main() {
	fmt.Println(SuccessCode)
	fmt.Println(ValidCode)
	fmt.Println(ServerCode)
}
```

这样比直接写 `0`、`7`、`8` 更容易理解。

## 3. 给自定义类型绑定方法

自定义类型可以绑定方法。

```go
package main

import "fmt"

type Code int

const SuccessCode Code = 0

func (c Code) Message() string {
	if c == SuccessCode {
		return "成功"
	}

	return "失败"
}

func main() {
	fmt.Println(SuccessCode.Message())
}
```

**重点：**给类型绑定方法后，数据和行为可以放在一起。

## 4. 自定义类型和原始类型比较

自定义类型和底层类型不是同一个类型，比较时通常要转换。

```go
package main

import "fmt"

type Code int

func main() {
	var code Code = 0
	var num int = 0

	fmt.Println(int(code) == num)
}
```

**注意：**`Code` 的底层是 `int`，但 `Code` 不是 `int`。

## 5. 类型别名

类型别名用 `=` 定义。

```go
type AliasCode = int
```

`AliasCode` 只是 `int` 的另一个名字，不是新类型。

```go
package main

import "fmt"

type AliasCode = int

func main() {
	var code AliasCode = 0
	var num int = 0

	fmt.Println(code == num)
	fmt.Printf("%T\n", code)
}
```

输出类型仍然是：

```text
int
```

## 6. 自定义类型 vs 类型别名

```go
type MyCode int
type AliasCode = int
```

区别：

| 写法 | 是否新类型 | 能否绑定方法 | 和原始类型比较 |
| --- | --- | --- | --- |
| `type MyCode int` | 是 | 可以 | 通常需要转换 |
| `type AliasCode = int` | 否 | 不可以 | 可以直接比较 |

**重点：**想表达业务含义，用自定义类型；只想换个名字，用类型别名。

## 7. 易错点

- **自定义类型是新类型**。
- **类型别名不是新类型，只是原类型的另一个名字**。
- **自定义类型可以绑定方法**。
- **类型别名不能绑定新方法**。
- **自定义类型和原始类型比较时，通常要显式转换**。
