1. == 和 equals( ) 方法有什么区别
	1. 基本数据类型 == 比较返回 true，引用数据类型比较返回 false，因为是不同的对象，内存地址不同
	2. 引用数据类型 重写了 equals( ) 比较返回true，没有重写 equals( ) 比较返回false，默认比较内存地址
2. 重写 equals 方法需要重写 hashcode 方法，为什么？
	1. Object 中的 hashCode 调用了一个（native）本地方法，返回了一个 int 类型的整数，这个整数可能是正数也可能是负数。
	2. 相等的值 hashCode 一定相同
	3. 不同的值 hashCode 也有可能相同
3. 如果不重写 hashcode，用 HashSet 会有什么bug？
	1. HashSet 通过比较 object当中HashCode方法进行比较去重来存数据，不重写会出现存相同值的情况

6. 数据库中索引有什么作用
	1. 快速定位数据，提高查询速度
	2. 主键索引可以保证数据唯一性
	3. 索引太多写入数据速度变慢
	4. 索引适合读多写少场景
7. Java内存溢出OOM是什么概念？有什么解决方法？
	1. **内存溢出（Out Of Memory Error）** 是Java程序运行时，JVM内存不足，无法为新对象分配内存时抛出的错误。
	2. 主要类型
		1. 堆溢出
		2. 栈溢出
		3. 元空间溢出
		4. GC超限
		5. 直接内存溢出
	3. 解决方法
		1. 方法一：扩大JVM运行内存
		2. 方法二：优化程序代码：及时释放资源、分页查询、使用线程池、用StringBuilder拼接
		3. 用java指令查情况
		4. 方法三：缓存缓冲流量
8. 乐观锁和悲观锁区别
	1. 悲观锁：假设冲突一定会发生，操作数据前先加锁，阻塞其他操作
		1. FOR UPDATE
		2. redis互斥锁
	2. 乐观锁：假设冲突很少发生，操作数据时不加锁，更新时检查是否被修改
		1. 版本号version
		2. 时间戳timestamp
		3. 库存 > 0
