# echo、tail、重定向与反引号

> `echo` 用于输出文本，`tail` 用于查看文件末尾，重定向符改变输出方向，反引号用于执行命令并替换为结果。

---

## 1. echo：输出文本

`echo` 是最基础的输出命令，常用于打印变量、写入文件或调试脚本。

### 基本语法

```bash
echo [选项] [字符串...]
```

### 常用选项

| 选项 | 作用 |
|------|------|
| `-n` | 不换行输出 |
| `-e` | 解释转义字符（如 `\n` 换行、`\t` 制表符） |

### 示例

```bash
# 输出普通文本
echo "Hello World"

# 输出不换行
echo -n "Loading..."

# 解释转义字符
echo -e "Line 1\nLine 2\tTabbed"

# 输出变量内容
name="Linux"
echo "Hello, $name"

# 输出当前时间
echo "Current time: $(date)"
```

> **注意**：`echo` 和 `printf` 的区别在于 `echo` 会自动换行，`printf` 更灵活但需要手动控制格式。

## 2. 反引号 `` ` ` ` 与 `$()`：命令替换

反引号或 `$()` 将其中的命令执行结果替换到当前命令中。

### 语法对比

```bash
# 反引号（旧写法，不推荐）
echo `date`

# $() 推荐写法，支持嵌套
echo $(date)

# 嵌套示例
echo "Today is $(date +%Y-%m-%d)"
```

### 示例

```bash
# 创建以当前日期命名的文件夹
mkdir backup_$(date +%Y%m%d)

# 将命令输出赋值给变量
files=$(ls -l)
echo "$files"

# 在字符串中嵌入命令结果
echo "There are $(ls | wc -l) files in this directory"
```

![[Pasted image 20260629191634.png]]

> **推荐**：始终使用 `$()` 而不是反引号，因为 `$()` 可以嵌套、可读性更好，且不容易与单引号混淆。

## 3. tail：查看文件末尾

`tail` 用于查看文件的最后几行，是查看日志文件的首选工具。

### 基本语法

```bash
tail [选项] [文件...]
```

### 常用选项

| 选项 | 作用 |
|------|------|
| `-n N` | 显示最后 N 行（默认 10 行） |
| `-f` | **实时追踪**（follow），当文件增长时持续输出新内容 |
| `-F` | 类似 `-f`，但如果文件被重命名或重新创建会继续跟踪 |
| `-c N` | 显示最后 N 个字节 |

### 示例

```bash
# 查看文件最后 10 行（默认）
tail log.txt

# 查看文件最后 50 行
tail -n 50 log.txt

# 简写：-n 50 可以写成 -50
tail -50 log.txt

# 实时查看日志（最常用！）
tail -f /var/log/nginx/access.log

# 同时查看多个日志文件
tail -f /var/log/nginx/*.log

# 实时查看并过滤关键词
tail -f app.log | grep "ERROR"
```

![[Pasted image 20260629193621.png]]

> **日志排查场景**：`tail -f` 配合 `grep` 是线上排查问题的黄金组合。例如：`tail -f /var/log/app.log | grep -i "error"` 可以实时过滤错误日志。

## 4. 重定向符

重定向符改变命令的输入/输出方向，将结果写入文件或从文件读取。

### 4.1 输出重定向

| 符号 | 名称 | 作用 |
|------|------|------|
| `>` | 覆盖写入 | 将输出写入文件，**覆盖原有内容** |
| `>>` | 追加写入 | 将输出追加到文件**末尾** |
| `2>` | 错误重定向 | 将错误输出写入文件 |
| `&>` | 全部重定向 | 将标准输出和错误输出都写入文件 |

```bash
# 覆盖写入（文件原有内容会被清空）
echo "Hello" > file.txt

# 追加写入（保留原有内容，在末尾添加）
echo "World" >> file.txt

# 将错误信息写入文件（覆盖）
ls /nonexistent 2> error.log

# 将输出和错误都写入同一个文件
command &> output.log

# 丢弃输出（写入 /dev/null）
command > /dev/null 2>&1
```

### 4.2 输入重定向

| 符号 | 作用 |
|------|------|
| `<` | 从文件读取输入 |
| `<<` | Here Document，从键盘输入多行文本 |

```bash
# 从文件读取输入并排序
sort < unsorted.txt

# 多行文本输入（Here Document）
cat << EOF
Line 1
Line 2
Line 3
EOF
```

![[Pasted image 20260629192000.png]]

> **重要区别**：`>` 会覆盖文件，`>>` 会追加。在脚本中误用 `>` 可能导致数据丢失！

## 5. 快速对比

| 需求 | 命令/符号 | 说明 |
|------|-----------|------|
| 输出文本 | `echo` | 基础输出，支持变量和转义 |
| 获取命令结果 | `$(cmd)` | 推荐写法，可嵌套 |
| 查看日志末尾 | `tail -n 20` | 查看最后 20 行 |
| 实时追踪日志 | `tail -f` | 日志监控必备 |
| 写入文件（覆盖） | `>` | 谨慎使用 |
| 写入文件（追加） | `>>` | 更安全 |
| 丢弃所有输出 | `> /dev/null 2>&1` | 后台任务常用 |

> **组合实战**：`echo "Backup completed at $(date)" >> backup.log` 将备份完成时间和日期追加到日志文件。
