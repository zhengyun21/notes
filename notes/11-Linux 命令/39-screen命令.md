# screen：会话管理器

> `screen` 是一个终端多路复用器，允许在单个 SSH 会话中创建多个窗口，即使 SSH 断开，后台任务也会继续运行。

---

## 为什么需要 screen

| 场景 | 问题 | screen 解决 |
|------|------|-------------|
| 执行长时间任务 | SSH 断开，任务终止 | 任务在 screen 会话中持续运行 |
| 多任务并行 | 需要多个终端窗口 | 一个 SSH 连接中管理多个窗口 |
| 查看实时日志 | 退出后日志不刷新 | 断开不影响正在运行的进程 |
| 远程部署 | 网络不稳定导致中断 | 断网后重连，恢复会话 |

---

## 基本用法

### 创建新会话

```bash
# 创建会话（默认名称）
screen

# 创建带名称的会话（推荐）
screen -S mysession
```

### 常用快捷键

> **注意**：screen 快捷键以 `Ctrl + a` 开头，松开后按下一个键。

| 快捷键 | 作用 |
|--------|------|
| `Ctrl + a` 后按 `d` | **分离会话**（detach），回到原终端，会话在后台运行 |
| `Ctrl + a` 后按 `c` | 创建新窗口 |
| `Ctrl + a` 后按 `n` | 切换到下一个窗口 |
| `Ctrl + a` 后按 `p` | 切换到上一个窗口 |
| `Ctrl + a` 后按 `数字` | 切换到指定窗口（如 `Ctrl + a` + `0`） |
| `Ctrl + a` 后按 `"` | 列出所有窗口 |
| `Ctrl + a` 后按 `k` | 关闭当前窗口 |
| `Ctrl + a` 后按 `?` | 显示所有快捷键帮助 |

---

## 会话管理

```bash
# 列出所有会话
screen -ls

# 重新连接到会话
screen -r 会话名或PID

# 如果会话已 attached，强制接管
screen -d -r 会话名

# 直接创建并运行命令
screen -S backup -dm bash -c 'tar -czvf backup.tar.gz /data/ && echo "Done"'

# 终止会话
screen -S 会话名 -X quit
```

---

## 完整工作流程示例

```bash
# 1. 创建命名会话
screen -S deploy

# 2. 在会话中执行长时间任务
./deploy.sh

# 3. 按 Ctrl + a，然后按 d，分离会话（任务继续在后台运行）

# 4. 之后重新连接服务器，恢复会话
screen -r deploy

# 5. 查看任务执行状态

# 6. 任务完成后，退出 screen（exit 或 Ctrl + d）
```

## 与 nohup 的对比

| 特性 | screen | nohup |
|------|--------|-------|
| 断网后恢复 | ✅ 可以重新接入 | ❌ 无法重新接入 |
| 多窗口管理 | ✅ 支持 | ❌ 不支持 |
| 查看实时输出 | ✅ 可以 | ❌ 只能看日志文件 |
| 使用复杂度 | 中等 | 简单 |
| 适用场景 | 长时间交互式任务 | 简单后台脚本 |

```bash
# nohup 简单示例（适合一次性后台任务）
nohup ./long_task.sh > output.log 2>&1 &
```

## 快速对比

| 需求 | 命令 |
|------|------|
| 创建会话 | `screen -S 名称` |
| 分离会话 | `Ctrl + a` 后按 `d` |
| 列出会话 | `screen -ls` |
| 恢复会话 | `screen -r 名称` |
| 创建新窗口 | `Ctrl + a` 后按 `c` |
| 切换窗口 | `Ctrl + a` 后按 `n` / `p` |
| 关闭窗口 | `Ctrl + a` 后按 `k` |
| 强制终止会话 | `screen -S 名称 -X quit` |

> **推荐**：凡是需要长时间运行的命令（如大数据处理、模型训练、系统部署），都建议在 `screen` 会话中执行，防止 SSH 意外断开导致任务中断。
