# iostat：I/O 性能监控

> `iostat` 用于监控系统的 CPU 使用情况和块设备的 I/O 统计信息，是排查磁盘性能瓶颈的常用工具。

---

## 基本语法

```bash
iostat [选项] [间隔] [次数]
```

---

## 常用选项

| 选项 | 作用 |
|------|------|
| `-c` | 只显示 CPU 统计 |
| `-d` | 只显示设备统计 |
| `-x` | 显示扩展统计（更详细，推荐） |
| `-k` | 以 KB/s 为单位显示 |
| `-m` | 以 MB/s 为单位显示 |
| `-t` | 显示时间戳 |
| `-p` | 显示设备和分区的统计 |
| `-y` | 跳过第一次统计（系统启动以来的累计值） |

---

## 示例

```bash
# 查看系统启动以来的 I/O 统计（一次性输出）
iostat

# 每2秒刷新一次，共刷新3次（最常用）
iostat -x 2 3

# 显示扩展统计（推荐）
iostat -x

# 只显示设备统计，以 MB/s 为单位
iostat -dm

# 显示每个分区的统计
iostat -p

# 显示 CPU 和设备（跳过第一次累计值）
iostat -xy 2 3
```

## 输出解读

### CPU 部分

| 字段 | 含义 |
|------|------|
| `%user` | 用户空间 CPU 使用率 |
| `%nice` | 带 nice 优先级的 CPU 使用率 |
| `%system` | 内核空间 CPU 使用率 |
| `%iowait` | **等待 I/O 的 CPU 时间**（重要！） |
| `%steal` | 被虚拟机偷走的 CPU 时间 |
| `%idle` | 空闲 CPU |

> **重点**：`%iowait` 高说明 CPU 大量时间花在等待磁盘 I/O，可能存在磁盘瓶颈。

### 设备部分（iostat -x）

| 字段 | 含义 | 说明 |
|------|------|------|
| `Device` | 设备名称 | 如 sda、sdb |
| `rrqm/s` | 每秒合并的读请求 | 合并越多，效率越高 |
| `wrqm/s` | 每秒合并的写请求 | — |
| `r/s` | 每秒读请求数 | IOPS 的读部分 |
| `w/s` | 每秒写请求数 | IOPS 的写部分 |
| `rkB/s` | 每秒读取的 KB | 吞吐量读部分 |
| `wkB/s` | 每秒写入的 KB | 吞吐量写部分 |
| `await` | **平均 I/O 等待时间**（毫秒） | **重要指标**，越低越好 |
| `r_await` | 读请求平均等待时间 | — |
| `w_await` | 写请求平均等待时间 | — |
| `svctm` | 平均服务时间 | 已废弃，不推荐参考 |
| `%util` | **设备利用率** | **重要！** 接近 100% 说明设备饱和 |

> **关键指标**：
> - `%util` > 90%：磁盘可能饱和，需要优化或升级
> - `await` > 50ms：I/O 延迟较高，可能磁盘性能不足或请求队列过长
> - `r/s + w/s`：即 IOPS，衡量磁盘随机读写能力

## 快速对比

| 需求 | 命令 |
|------|------|
| 查看 I/O 概况 | `iostat` |
| 实时监控 I/O | `iostat -x 2` |
| 查看 CPU 等待 I/O | `iostat -c`（看 %iowait） |
| 查看设备利用率 | `iostat -x`（看 %util） |
| 以 MB/s 查看 | `iostat -xm` |

---

## 与 sar -d 的区别

| 特性 | iostat | sar -d |
|------|--------|--------|
| 实时显示 | ✅ | ✅ |
| 历史数据 | ❌ | ✅（保存到日志） |
| 详细程度 | 更详细（-x 扩展） | 基础统计 |
| 使用场景 | 实时排查 | 趋势分析 |

> **推荐**：实时排查用 `iostat -x 2`，长期监控用 `sar -d`。
