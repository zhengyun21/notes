# ps：查看进程状态

> `ps`（Process Status）用于查看当前系统上的进程信息。它是静态快照，不会实时刷新。

---

## 基本语法

```bash
ps [选项]
```

---

## 常用选项组合

| 命令 | 作用 |
|------|------|
| `ps aux` | 显示所有用户的所有进程（最常用） |
| `ps -ef` | 显示所有进程的完整格式 |
| `ps -l` | 显示长格式 |
| `ps -ejH` | 显示进程树 |
| `ps -e --forest` | 以树状结构显示进程关系 |
| `ps -u 用户名` | 显示指定用户的进程 |
| `ps -p PID` | 显示指定 PID 的进程 |
| `ps -C 进程名` | 按进程名查找 |

---

## 示例

```bash
# 查看所有进程（最常用）
ps aux

# 查看指定进程名的进程
ps -C nginx

# 查看指定用户的进程
ps -u root

# 查看指定 PID 的进程
ps -p 1234

# 查看进程树
ps -ejH

# 查看进程并按 CPU 占用排序
ps aux --sort=-%cpu | head -n 20

# 查看进程并按内存占用排序
ps aux --sort=-%mem | head -n 20

# 查看进程并过滤
ps aux | grep nginx
```

## 输出字段解读

### `ps aux` 输出

| 字段 | 含义 |
|------|------|
| `USER` | 进程所有者 |
| `PID` | 进程 ID |
| `%CPU` | CPU 占用百分比 |
| `%MEM` | 内存占用百分比 |
| `VSZ` | 虚拟内存大小（KB） |
| `RSS` | 实际物理内存（KB） |
| `TTY` | 终端（`?` 表示后台进程） |
| `STAT` | 进程状态 |
| `START` | 启动时间 |
| `TIME` | 累计 CPU 时间 |
| `COMMAND` | 启动命令 |

### 进程状态（STAT）

| 状态 | 含义 |
|------|------|
| `R` | 运行中（Running） |
| `S` | 睡眠中（Sleeping，可中断） |
| `D` | 不可中断睡眠（Disk Sleep，通常等待 I/O） |
| `T` | 停止（Stopped） |
| `Z` | 僵尸进程（Zombie） |
| `I` | 空闲（Idle） |
| `+` | 前台进程 |
| `l` | 多线程 |
| `s` | 会话领导者 |
| `<` | 高优先级 |
| `N` | 低优先级 |

## 与 top 的区别

| 特性 | ps | top |
|------|-----|------|
| 更新方式 | 静态快照 | 实时动态刷新 |
| 交互性 | 非交互 | 可交互操作 |
| 排序 | 输出后排序 | 按列排序 |
| 查看历史 | 只能看当前 | 可以看历史趋势 |
| 使用场景 | 脚本中获取进程信息 | 实时监控系统 |

---

## 快速对比

| 需求 | 命令 |
|------|------|
| 查看所有进程 | `ps aux` |
| 查看进程树 | `ps -ejH` |
| 按 CPU 排序 | `ps aux --sort=-%cpu` |
| 按内存排序 | `ps aux --sort=-%mem` |
| 查找特定进程 | `ps aux \| grep nginx` |
| 查看指定 PID | `ps -p 1234` |
| 查看进程完整命令 | `ps -ef` |

> **注意**：`ps aux` 和 `ps -ef` 的输出格式略有不同，`aux` 显示 `%CPU`/`%MEM` 更直观，`-ef` 显示 `PPID`（父进程 ID）更完整。根据需要选择。
