# 权限信息：rwx 解读

> Linux 中每个文件/目录都有权限属性，控制谁可以读、写、执行。理解 `rwx` 是权限管理的第一步。

---

## 1. 权限信息结构

使用 `ls -l` 命令时，第一列就是权限信息：

```bash
$ ls -l /etc/passwd
-rw-r--r-- 1 root root  2875 Jul  4 10:00 /etc/passwd
```

权限列 `-rw-r--r--` 共 10 位，分为四段：

| 位置 | 含义 | 示例中的值 |
|------|------|------------|
| 第 1 位 | 文件类型 | `-` 表示普通文件，`d` 表示目录，`l` 表示软链接 |
| 第 2-4 位 | 所有者（Owner）权限 | `rw-` |
| 第 5-7 位 | 所属组（Group）权限 | `r--` |
| 第 8-10 位 | 其他用户（Others）权限 | `r--` |

## 2. rwx 的含义

| 权限 | 对文件 | 对目录 |
|------|--------|--------|
| `r`（read） | 读取文件内容 | 列出目录中的文件列表（`ls`） |
| `w`（write） | 修改文件内容 | 在目录中创建、删除、重命名文件 |
| `x`（execute） | 执行文件（脚本/程序） | 进入目录（`cd`） |

### 权限组合示例

| 权限 | 数字 | 文件效果 | 目录效果 |
|------|------|----------|----------|
| `rwx` | 7 | 可读可写可执行 | 可进入、可列出、可创建/删除 |
| `rw-` | 6 | 可读可写 | 可列出，但不能进入 |
| `r-x` | 5 | 可读可执行 | 可进入、可列出，但不能创建/删除 |
| `r--` | 4 | 只读 | 只能列出文件名，不能进入 |
| `-wx` | 3 | 可写可执行（罕见） | 可进入、可创建，但不能列出 |
| `-w-` | 2 | 只写（罕见） | 几乎无用 |
| `--x` | 1 | 只可执行（罕见） | 可进入，但不能列出 |
| `---` | 0 | 无任何权限 | 无任何权限 |

> **重要**：对于目录，`x` 权限比 `r` 更重要。没有 `x` 权限，即使有 `r` 权限也无法 `cd` 进入该目录。

## 3. 特殊权限位

除了基本的 `rwx`，还有三种特殊权限：

| 符号 | 名称 | 作用 |
|------|------|------|
| `s` | SUID（Set User ID） | 执行时以文件所有者身份运行（常用于 `passwd` 命令） |
| `s` | SGID（Set Group ID） | 执行时以文件所属组身份运行；对新创建的文件继承目录组 |
| `t` | Sticky Bit | 目录中，只有文件所有者或 root 才能删除文件（常用于 `/tmp`） |

### 特殊权限位置

- SUID 在所有者的 `x` 位：`rwsr-xr-x`（数字表示 4755）
- SGID 在所属组的 `x` 位：`rwxr-sr-x`（数字表示 2755）
- Sticky Bit 在其他人的 `x` 位：`rwxrwxrwt`（数字表示 1755）

```bash
# 查看 /tmp 目录的权限（带有 Sticky Bit）
ls -ld /tmp
# drwxrwxrwt  14 root root 4096 Jul  5  2024 /tmp
```

---

## 4. 快速判断权限

```bash
# 查看当前目录下所有文件的权限
ls -l

# 查看某个目录本身的权限（不是目录内的文件）
ls -ld mydir

# 查看自己的身份
id
```

---

## 5. 权限速查表

| 权限字符串 | 数字 | 常见用途 |
|------------|------|----------|
| `-rw-r--r--` | 644 | 普通文件（所有者可读写，其他人只读） |
| `-rw-------` | 600 | 敏感文件（如私钥） |
| `-rwxr-xr-x` | 755 | 可执行脚本/程序 |
| `-rwx------` | 700 | 私有可执行文件 |
| `drwxr-xr-x` | 755 | 普通目录 |
| `drwxrwxrwx` | 777 | 完全开放（不推荐，除非特殊需求） |
| `drwxr-x---` | 750 | 组内共享目录 |
| `drwxrwxrwt` | 1777 | 公共临时目录（如 /tmp） |
