# top：实时系统监控

> `top` 是 Linux 下最经典的实时系统性能监控工具，动态显示系统资源使用情况和进程信息。更详细的解读可参考「40-系统运维常见问题」。

---

## 基本用法

```bash
top
```

运行后进入交互式界面，按 `q` 退出。

## 界面分区

| 区域 | 内容 | 说明 |
|------|------|------|
| **上半部分** | 系统整体概况 | 运行时间、负载、CPU、内存 |
| **下半部分** | 进程列表 | 按默认排序显示各进程详情 |

---

## 常用交互命令

| 按键 | 作用 |
|------|------|
| `q` | 退出 top |
| `Space` | 立即刷新 |
| `k` | 输入 PID 后终止进程 |
| `r` | 修改进程优先级（renice） |
| `M` | 按内存使用率排序 |
| `P` | 按 CPU 使用率排序 |
| `T` | 按累计 CPU 时间排序 |
| `1` | 显示每个 CPU 核心的使用情况 |
| `c` | 切换显示完整命令路径 |
| `u` | 按用户名过滤进程 |
| `H` | 显示线程而不是进程 |
| `i` | 只显示正在运行的进程 |
| `d` | 修改刷新间隔（秒） |
| `W` | 保存当前配置到 ~/.toprc |
| `h` | 显示帮助 |

---

## 常用启动选项

| 选项 | 作用 |
|------|------|
| `-d 秒` | 设置刷新间隔 |
| `-n 次` | 刷新指定次数后退出 |
| `-p PID` | 只监控指定进程 |
| `-u 用户` | 只显示指定用户的进程 |
| `-b` | 批处理模式（用于脚本输出） |

```bash
# 只监控 PID 1234
top -p 1234

# 批处理模式，显示3次后退出
top -b -n 3

# 只显示 root 用户的进程
top -u root

# 每2秒刷新一次
top -d 2
```

---

## 输出重定向到文件

```bash
# 将 top 输出保存到文件（用于分析）
top -b -n 5 > top.log
```

## 与 htop 的对比

| 特性 | top | htop |
|------|-----|------|
| 是否预装 | 是 | 需安装 |
| 界面 | 文字 | 彩色 + 柱状图 |
| 鼠标支持 | 否 | 是 |
| 进程搜索 | 支持过滤 | 支持实时搜索 |
| 树形显示 | 需按键 | 支持 |
| 推荐 | 基础监控 | 日常使用 |

```bash
# 安装 htop
sudo apt install htop    # Ubuntu/Debian
sudo yum install htop    # CentOS/RHEL
```

## 快速对比

| 需求 | 操作 |
|------|------|
| 启动 top | `top` |
| 按 CPU 排序 | `P`（大写） |
| 按内存排序 | `M`（大写） |
| 显示每个 CPU | `1` |
| 终止进程 | `k` → 输入 PID → 回车 |
| 退出 | `q` |
| 只监控某进程 | `top -p PID` |
| 批处理输出 | `top -b -n 5 > log.txt` |

> **提示**：`top` 是排查系统性能问题的第一工具。发现 CPU 或内存异常时，先用 `top` 定位具体进程，再深入分析。详见「40-系统运维常见问题」中的详细解读。
