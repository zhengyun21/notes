# su、sudo 与 sudo su：用户切换详解

> 这是 `su` 和 `sudo` 命令的深入补充，包括更常用的快捷切换方式。

---

## 1. su：切换用户

| 命令 | 作用 |
|------|------|
| `su` | 切换到 root，保留当前环境变量 |
| `su -` | 完全切换到 root，加载 root 的环境变量（推荐） |
| `su - 用户名` | 完全切换到指定用户 |
| `exit` | 退出切换，回到原用户 |

```bash
# 完全切换到 root（最常用）
su -

# 切换到 tomcat 用户
su - tomcat

# 回到原用户
exit
```

## 2. sudo：执行单条高权限命令

| 命令 | 作用 |
|------|------|
| `sudo 命令` | 以 root 权限执行单条命令 |
| `sudo -i` | 切换到 root 用户（等价于 `su -`） |
| `sudo -s` | 以 root 身份打开 shell（不加载 root 环境） |
| `sudo -u 用户 命令` | 以指定用户身份执行命令 |

```bash
# 以 root 权限安装软件
sudo apt install nginx

# 切换到 root 用户
sudo -i

# 以 www-data 用户运行脚本
sudo -u www-data python app.py
```

## 3. sudo su / sudo -i / su - 的区别

| 命令 | 是否需要 root 密码 | 是否加载 root 环境 | 是否记录审计日志 | 推荐场景 |
|------|-------------------|-------------------|------------------|----------|
| `su -` | 是 | 是 | 否 | 知道 root 密码时使用 |
| `sudo -i` | 否（用自己的密码） | 是 | 是 | **推荐** 获取 root shell |
| `sudo su -` | 否 | 是 | 是 | 同上，较老的习惯写法 |
| `sudo -s` | 否 | 否 | 是 | 临时 root shell，保留当前环境 |

```bash
# 最常用的三种方式：
sudo -i          # 推荐：完全切换到 root
sudo su -        # 效果同上（旧写法）
sudo -s          # 以 root 身份运行 shell，但保留当前环境
```

> **建议**：现代 Linux 发行版推荐使用 `sudo -i` 获取 root shell，而不是 `sudo su -`，更简洁且语义明确。

---

## 4. 快速对比

| 需求 | 命令 |
|------|------|
| 执行单条 root 命令 | `sudo 命令` |
| 获取 root shell（推荐） | `sudo -i` |
| 获取 root shell（老写法） | `sudo su -` |
| 临时 root shell（保留环境） | `sudo -s` |
| 切换到其他用户 | `su - 用户名` |
| 回到普通用户 | `exit` 或 `logout` |
| 查看当前用户 | `whoami` |
| 查看用户 ID 信息 | `id` |
