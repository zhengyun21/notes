# ln：创建软链接与硬链接

> `ln`（Link）用于创建文件链接，分为**软链接**（Symbolic Link，相当于快捷方式）和**硬链接**（Hard Link）两种。

---

## 1. 软链接（Symbolic Link）

软链接是一个独立的文件，指向另一个文件或目录的路径。类似于 Windows 的快捷方式。

### 特点

- 可以跨越文件系统
- 可以对目录创建软链接
- 删除原文件后，软链接失效（称为" dangling link "）
- 软链接有自己的 inode 号
- 权限显示为 `lrwxrwxrwx`

### 基本语法

```bash
ln -s 目标文件/目录 链接名
```

### 示例

```bash
# 给文件创建软链接
ln -s /usr/local/bin/python3.9 /usr/bin/python3

# 给目录创建软链接
ln -s /export/server/apache-tomcat-10.0.27 /export/server/tomcat

# 创建软链接后查看
ls -l /export/server/tomcat
# 输出：lrwxrwxrwx 1 root root 38 Jul  5 10:00 tomcat -> /export/server/apache-tomcat-10.0.27

# 删除软链接（注意：不要加 /，否则会删除原目录内容！）
rm /export/server/tomcat
```

> ⚠️ **警告**：删除软链接时，`rm 链接名` 即可。**不要写成 `rm 链接名/`**，否则系统会把原目录内容当成目标删除！

## 2. 硬链接（Hard Link）

硬链接是文件系统层面的链接，与原文件共享同一个 inode 和数据块。

### 特点

- 不能跨越文件系统
- 不能对目录创建硬链接（防止形成循环）
- 删除原文件后，硬链接仍然可用（数据不会丢失）
- 硬链接与原文件有相同的 inode 号
- 只有所有硬链接和原文件都被删除后，数据才真正被释放

### 基本语法

```bash
ln 原文件 硬链接名
```

### 示例

```bash
# 创建硬链接
ln file.txt file_hard.txt

# 查看 inode 号，发现两者相同
ls -li file.txt file_hard.txt

# 删除原文件，硬链接仍然可用
rm file.txt
cat file_hard.txt   # 内容还在
```

---

## 3. 软链接 vs 硬链接

| 特性 | 软链接 | 硬链接 |
|------|--------|--------|
| 命令 | `ln -s` | `ln` |
| 文件类型 | `l`（link） | 普通文件 `-` |
| 能否跨文件系统 | ✅ 可以 | ❌ 不可以 |
| 能否链接目录 | ✅ 可以 | ❌ 不可以 |
| 原文件删除后 | 链接失效 | 数据仍然可用 |
| inode 号 | 不同 | 相同 |
| 占用空间 | 极小（存储路径） | 无额外数据空间 |
| 使用场景 | 快捷方式、版本切换 | 数据备份保护 |

---

## 4. 常见应用场景

```bash
# 场景1：软件版本管理（通过切换软链接切换版本）
ln -s /export/server/jdk1.8.0_351 /export/server/jdk
# 升级时只需重建软链接：
rm /export/server/jdk
ln -s /export/server/jdk-17 /export/server/jdk

# 场景2：配置文件统一管理
ln -s /home/user/configs/.bashrc /home/user/.bashrc

# 场景3：多入口访问同一目录
ln -s /var/www/html /var/www/web
```

---

## 5. 快速参考

| 需求 | 命令 |
|------|------|
| 创建软链接 | `ln -s 目标 链接名` |
| 创建硬链接 | `ln 目标 链接名` |
| 查看软链接指向 | `ls -l 链接名` |
| 安全删除软链接 | `rm 链接名`（不要加 /） |
| 查看 inode 号 | `ls -li 文件` |
| 查找所有软链接 | `find . -type l` |
| 查找 dangling 链接 | `find . -type l ! -exec test -e {} \; -print` |

> **重要提示**：在脚本中使用软链接指向的目录时，注意相对路径的解析。建议创建软链接时使用绝对路径，避免因工作目录变化导致链接失效。
