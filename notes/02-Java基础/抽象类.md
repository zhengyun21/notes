1. 概述
	1. 一种特殊类，不能实例化，但可以作为其他类的父类
	2. 提供一种抽象模板，用来定义子类的行为和属性
2. 和普通类区别
	1. 不能被实例化：不能直接创建抽象类的对象，只能通过继承抽象类来创建子类的对象。
	2. 可以包含抽象方法：抽象方法是没有具体实现的方法，子类必须实现抽象方法。
	3. 可以包含非抽象方法：抽象类中可以包含普通的方法，子类可以直接继承和使用这些方法。
	4. 可以包含成员变量和构造方法：与普通类一样，抽象类可以有成员变量和构造方法。**构造方法让子类在创建对象时，先初始化抽象父类里的成员变量、执行父类的初始化逻辑。**

3. 具体表现
``` java
 abstract class AbstractClass{
	 //成员变量
	 private String name;
	 private int age;
 
	 //构造方法
	 public AbstractClass(String name,int age){
		 this.name = name;
		 this.age = age;
	 }
 
	 //抽象方法(没有实现逻辑)
	 public abstract void abstractMethod();
	 
	 //普通方法
	 public void Method(){
		 //实现逻辑
	 }
 }
```

4. 抽象类作为父类时，需要通过继承实现抽象类方法
	1. ❗❗❗**注意**
		1. 这个super必须写，因为java会在你调用构造方法时，保底都调用一次super的无参构造方法来执行父类构造，因为**AbstractClass**没有无参构造，如果不调用super的无参构造这里就会报错。
			1.     - **子类构造必须调用父类构造**，这是 Java 强制规则。
				- **你不写，Java 自动加 `super()`**。
				- **`this` 是调用自己，`super` 是调用父类，不能互相替代。**
``` java
	class child extends AbstractClass{
		String bread;
		
		//子类构造方法
		public child(String name,int age,String bread){
			//必须调用父类构造方法！
			super(name,age);
			this.bread = bread;
		}
		
		@Override  
		public void abstractMethod() {  
			//重写逻辑
			...
		}
	}
```
